# 电子钟软件结构说明

## 1. 当前分层

### `bsp`

职责：

- 管具体板级资源
- 例如 LED、按键、串口引脚映射

当前文件：

- `board_config.h`
- `bsp_led.c`

### `drivers`

职责：

- 管通用驱动逻辑
- 不直接掺业务规则

当前文件：

- `drv_systick.c`
- `drv_key.c`
- `drv_debug_uart.c`

### `application`

职责：

- 管电子钟自己的业务状态
- 例如当前模式、时间更新、按键事件处理

当前文件：

- `app_clock.c`

### `User`

职责：

- 工程入口
- 中断转发

当前文件：

- `main.c`
- `stm32f10x_it.c`

## 2. 当前执行流

1. 上电进入 `main`
2. `App_Clock_Init()` 初始化 LED、按键、串口、SysTick
3. `SysTick_Handler()` 每 1ms 进入一次
4. 中断里更新毫秒计数，并调用按键扫描
5. 主循环一直调用 `App_Clock_Task()`
6. 业务层读取按键事件、更新时间、控制 LED、打印日志

## 3. 这套结构为什么适合电赛训练

1. 代码不会全塞进 `main.c`
2. 后面加 LCD、RTC、蜂鸣器时有明确落点
3. 便于定位问题是“板级问题、驱动问题还是业务问题”
4. 后续迁移到采集系统、频率计也能复用

## 4. 本轮阅读后结论

### 4.1 已确认的硬件配置

基于 `李惠忠老师一.pptx`、`电子钟_需求拆解.md`、`clock_board_config.yaml` 以及屏幕配套资料，当前电子钟项目可直接确认的硬件资源如下：

1. 主控：`STM32F103VET6`，对应 `STM32F103VE` 高密度器件配置。
2. 调试串口：`USART1`，`PA9/PA10`，板载 `CH340`。
3. RTC：使用 STM32 内部 `RTC`，`PC14/PC15` 接 `32.768kHz`，后备电池为 `CR1220`。
4. 液晶：`3.2 寸 320x240`，控制器兼容 `ST7789V / ILI9341`，总线为 `FSMC 8080 16bit`。
5. 触摸：`XPT2046`，`PE0/PE2/PE3/PE4/PD13`，属于最终可替代多按键的输入资源。
6. 板载机械按键：
   - `KEY1 = PA0`
   - `KEY2 = PC13`
7. 触摸按键：`T-PAD = PA1`，为电容触摸输入，不是普通机械按键。
8. 蜂鸣器：`BEEP = PA8`，已知控制脚，极性待验证。
9. 板载 RGB LED：
   - `RED = PB5`
   - `GREEN = PB0`
   - `BLUE = PB1`

### 4.2 当前工程与资料的冲突

1. 当前 Keil 工程目标器件还是 `STM32F103C8`，启动文件是 `startup_stm32f10x_md.s`，与资料中的 `STM32F103VET6/VE` 不一致。
2. 当前 `board_config.h` 把 `PC13` 当作 LED，但硬件整理资料显示 `PC13` 更应作为 `KEY2`；真正板载 LED 应使用 `PB5/PB0/PB1`。
3. 当前按键驱动只有单键模型，不支持后续电子钟正式菜单。
4. 当前时间源仍是软件变量累加，不满足 `RTC + 断电走时` 目标。

### 4.3 目前仍存在的硬件缺口

1. `KEY3 / KEY4` 没有在当前板卡资料中找到明确的板载机械按键定义。
2. 最终若题目需要四个独立逻辑按键，应改为：
   - 使用触摸屏区域实现逻辑按键，或
   - 增加扩展输入硬件。
3. `BEEP` 的有效电平没有在当前资料中确认，上板前不应直接假设蜂鸣器极性。
4. 触摸屏虽然已有接口资料，但当前训练工程还没有引入 LCD / Touch 驱动，现阶段只能先做“逻辑键接口预留 + 物理按键代用”。

## 5. 本轮重构与实现计划

### 5.1 设计原则

1. 继续保留 `bsp / drivers / application / User` 分层。
2. 不提前引入 LCD 和触摸重驱动，先把 `RTC + 多输入菜单 + 真实板级映射` 跑通。
3. 对最终会切到触摸屏的输入，先定义清晰的逻辑动作，再用现有物理按键临时代用。

### 5.2 代码修改计划

1. 修正板级配置：
   - 把 LED 改为指南者板载 `RGB LED`
   - 把按键改为 `KEY1=PA0`、`KEY2=PC13`
2. 重构按键驱动：
   - 支持多按键编号
   - 支持 `短按 / 长按 / 长按连发`
   - 为后续触摸逻辑键保留映射空间
3. 新增 `RTC` 驱动层：
   - 优先使用硬件 RTC
   - 若 LSE/后备域初始化失败，自动降级为软件时基，避免程序卡死
4. 重构电子钟应用层：
   - 运行态
   - 改时态：`时 / 分 / 秒`
   - 闹钟设置态：`时 / 分`
   - `10s` 无操作返回
   - 闹钟使能与到点状态管理
5. 更新 Keil 工程：
   - 目标器件改为 `STM32F103VE`
   - 启动文件改为 `startup_stm32f10x_hd.s`
   - 新增源文件加入工程

### 5.3 本轮暂不直接落地的内容

1. LCD 界面显示
2. 触摸屏实际采样驱动
3. 蜂鸣器真实响声输出
4. 红外遥控

原因不是这些内容不重要，而是当前最先要解决的是：
`工程目标器件错误 + 板级映射错误 + 单键模型不足 + 没有 RTC`

# 电子钟软件结构说明

## 1. 当前分层

### `bsp`

职责：

- 管具体板级资源
- 例如 LED、按键、串口引脚映射

当前文件：

- `board_config.h`
- `bsp_led.c`

### `drivers`

职责：

- 管通用驱动逻辑
- 不直接掺业务规则

当前文件：

- `drv_systick.c`
- `drv_key.c`
- `drv_debug_uart.c`

### `application`

职责：

- 管电子钟自己的业务状态
- 例如当前模式、时间更新、按键事件处理

当前文件：

- `app_clock.c`

### `User`

职责：

- 工程入口
- 中断转发

当前文件：

- `main.c`
- `stm32f10x_it.c`

## 2. 当前执行流

1. 上电进入 `main`
2. `App_Clock_Init()` 初始化 LED、按键、串口、SysTick
3. `SysTick_Handler()` 每 1ms 进入一次
4. 中断里更新毫秒计数，并调用按键扫描
5. 主循环一直调用 `App_Clock_Task()`
6. 业务层读取按键事件、更新时间、控制 LED、打印日志

## 3. 这套结构为什么适合电赛训练

1. 代码不会全塞进 `main.c`
2. 后面加 LCD、RTC、蜂鸣器时有明确落点
3. 便于定位问题是“板级问题、驱动问题还是业务问题”
4. 后续迁移到采集系统、频率计也能复用

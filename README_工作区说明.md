# 个人软件实验区说明

这个目录不是资料原件区，而是你后面真正做题、写代码、留痕、整理成果的主工作区。

## 目录结构

- `01_需求与题目分析/`
  - 题目要求
  - 需求拆解
  - 输入输出接口清单
- `02_方案设计/`
  - 软件结构
  - 状态机
  - 引脚分配
  - 调试策略
- `03_软件工程/`
  - 电子钟训练工程
  - 后续其它题目的主工程
- `04_训练与调试记录/`
  - 训练日志
  - 环境安装记录
  - 问题排查
  - 联调记录
  - 测试证据
- `05_仿真分析/`
  - 与正式题目直接相关的仿真和推导
- `06_工具脚本/`
  - 启动脚本
  - 辅助命令
- `07_阶段成果/`
  - 阶段报告
  - 演示素材
- `08_硬件接口资料/`
  - 原理图
  - 引脚对应关系
  - 接线说明

## 当前重点

你现在最常用的地方只有三个：

1. `03_软件工程/电子钟训练工程/`
2. `02_方案设计/`
3. `04_训练与调试记录/`

## 当前原则

1. 题目分析先写到 `01`。
2. 结构和状态机先写到 `02`。
3. 代码只写到 `03`。
4. 每次训练结束必须在 `04` 留痕。

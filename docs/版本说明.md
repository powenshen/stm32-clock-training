# 版本说明

本仓库使用单仓库多分支管理电子钟训练项目，当前约定如下：

- `main`：当前稳定主线，保存当前训练代码与文档。
- `study/recomment`：去掉 AI 注释后，手工重新注释和重学代码的学习分支。
- `feat/software-complete`：在不依赖最终硬件联调结论的前提下，完善软件需求的分支。
- `feat/hardware-integrated`：结合真实开发板、外设接线和联调结果后的分支。

建议里程碑：

- `v0-current`：当前训练工程初始导入版本。
- `v1-study-comments`：学习注释版。
- `v2-software-complete`：软件需求完善版。
- `v3-hardware-integrated`：硬件联调版。

建议开发顺序：

1. `main` 保持可编译、可回退。
2. 学习注释工作在 `study/recomment` 上进行。
3. 软件功能完善先在 `feat/software-complete` 上推进。
4. 拿到真实板级信息后，再在 `feat/hardware-integrated` 上做硬件适配与联调。

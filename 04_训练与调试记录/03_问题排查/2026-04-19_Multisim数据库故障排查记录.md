# Multisim 数据库故障排查记录

## 时间

- 日期：2026-04-19
- 场景：训练块 2 安装与环境配置阶段
- 故障现象：打开 Multisim 后提示“主数据库、用户数据库、企业数据库都不可用”

## 已确认事实

1. 当前安装版本为 `Circuit Design Suite 14.3.2`。
2. 本机该版本的主数据库并不是常见说法里的 `master.mdb`，而是以下文件：
   - `C:\ProgramData\National Instruments\Circuit Design Suite\14.3\database\MSCOMP_S.PRD`
   - `C:\ProgramData\National Instruments\Circuit Design Suite\14.3\database\CPCOMP_S.PRJ`
   - `C:\Program Files (x86)\National Instruments\Circuit Design Suite 14.3\database\UsrCompTemplate_S.usr`
   - `C:\Users\powen\AppData\Roaming\National Instruments\Circuit Design Suite\14.3\database\UsrComp_S_powen.usr`
3. 上述数据库文件实际存在，不是简单的“文件丢失”。
4. DAO 组件可创建，说明系统里的 Jet/DAO 不是完全损坏状态。
5. Multisim 进程存在卡住现象，关闭后数据库目录内没有持续残留的 `.ldb` 锁文件。
6. `C:\Users\powen\AppData\Roaming\National Instruments\Circuit Design Suite\14.3\database` 在文件系统层面并没有真实的只读属性。
7. 当 Multisim 成功启动时，会同时产生以下锁文件：
   - `C:\ProgramData\National Instruments\Circuit Design Suite\14.3\database\MSCOMP_S.ldb`
   - `C:\ProgramData\National Instruments\Circuit Design Suite\14.3\database\CPCOMP_S.ldb`
   - `C:\Users\powen\AppData\Roaming\National Instruments\Circuit Design Suite\14.3\database\UsrComp_S_powen.ldb`

## 本次排查动作

1. 检查 Multisim 安装目录、数据库目录、注册表数据库路径。
2. 检查数据库文件是否存在、大小是否正常。
3. 检查数据库目录权限。
4. 关闭卡住的 `multisim.exe` 进程并复查锁文件。
5. 尝试使用本地 MSI 缓存修复数据库组件，失败。
6. 尝试使用 `NI Package Manager` 官方 `repair` 命令修复 `database/core/shared/pro` 等组件，失败。
7. 读取 `NI Package Manager` 日志定位失败阶段。
8. 对 `C:\ProgramData\National Instruments\MDF\ProductCache\...` 做写入自检，确认当前上下文无法写入。

## 关键结论

1. “缺少 `master.mdb/sqlite/ini`”不是本机这套 `14.3.2` 安装的核心证据，因为该版本本来就主要使用 `PRD/PRJ/USR` 数据库文件。
2. 问题更像是 NI 整套安装链路的权限/修复链路异常，而不是单个数据库文件没装上。
3. `NI Package Manager` 日志显示：
   - 调用已安装包数据库时出现 `sqlite3_open_v2 error code 14`
   - 重新缓存到 `MDF\ProductCache` 时失败
   - 随后 MSI 修复统一返回 `1625`
4. 当前环境下无法写入 `C:\ProgramData\National Instruments\MDF\ProductCache\...`，这会直接阻断官方修复流程。
5. 资源管理器里文件夹“只读”勾选自动恢复，不等于该目录真的存在只读属性；`attrib` 检查没有发现目录级 `R` 标记。
6. 你观察到“多开时偶尔有一个进程能用”，与锁文件现象一致，更像启动阶段的数据库初始化竞争，而不是数据库永久损坏。

## 对训练的影响

1. 无法正常浏览和调用 Multisim 主元件库。
2. 用户数据库、企业数据库不可用，无法稳定保存或调用自定义器件。
3. `Multisim M1/M2` 训练会被阻塞，近期不适合继续深入仿真任务。

## 建议处理顺序

1. 使用真正的 Windows 管理员权限启动 `NI Package Manager`。
2. 在图形界面中优先修复以下包：
   - `NI Circuit Design Suite Master Database`
   - `NI Circuit Design Suite 14.3.2 Core`
   - `NI Circuit Design Suite 14.3.2 Education`
   - `NI Circuit Design Suite 14.3.2 Pro`
3. 若图形界面修复仍失败，执行卸载后重新安装，并确保整个安装过程使用管理员权限。
4. 若机器属于学校实验室统一环境，需要联系管理员处理 `ProgramData\National Instruments\MDF\ProductCache` 写权限或安装策略问题。
5. 在根因未修复前，优先采用“只保留一个进程、失败后清理 `.ldb` 锁文件再重启”的临时工作流。
